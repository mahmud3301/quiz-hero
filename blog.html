What is the difference between map, filter, forEach and find?

map is used to transform elements, filter is used to select elements, forEach is used to iterate over elements and execute a function, and find is used to find a specific element.


What is the difference between local storage, session storage and cookies?

localStorage and sessionStorage are similar, but differ in their scope of data storage and data accessibility. Cookies have a shorter lifespan and smaller data storage limit, but can be accessed by server-side code and restricted to certain paths or subdomains. Each one has its own use cases and benefits depending on the specific requirements of the application.


What is an event loop and how does it work?

The event loop is a mechanism that enables asynchronous and non-blocking processing of code.

The event loop works by continuously checking two queues: the call stack and the message queue. The call stack is a data structure that stores a list of function calls that are currently being executed. When a function is called, it is added to the call stack, and when it returns, it is removed from the stack


How did the quizTimer function work?

This function implements a timer for a quiz, and updates the displayed time every second while also changing the color of the displayed time after 60 seconds


How did the chooseQuiz function work?

The chooseQuiz() function takes an index and a given answer as parameters and updates an array called answers with a new object that contains the quiz data and the given answer for that quiz. If an object with the same ID as the quiz being answered already exists in the answers array, the function updates the answer for that object. If no such object exists, it adds a new object to the answers array. Finally, the displayAnswers() function is called to update the user interface with the updated answers array.
